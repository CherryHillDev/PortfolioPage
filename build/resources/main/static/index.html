1. 개발 환경

  • Spring 5.2
  • JDK 11
  • Maven 2.9
  • Tomcat 9
  • Mybatis
  • MySQL 8.0.20 (RDS)
  • RHEL 8.3 (EC2)
  • Bootstrap 4
  • Jquery 3

2. 파일 구조

- 게시글, 댓글, 회원의 3개 테이블을 생성
- 기본키들은 자동 증가되는 int 타입
- 생성, 수정시간은 데이터의 생성, 수정 시 자동으로 정해지게 함
- 댓글 테이블은 그룹, 그룹 내 순서 컬럼을 만들어 대댓글을 구분해 저장할 수 있게 함

3. 데이터베이스 구조

4. 주요 내용

  • 회원가입 유효성 검사
    • 키가 입력될 때마다 검사해서, 유효함을 바로 알 수 있게 함
    아이디는 키가 입력될 때마다 ajax로 중복 여부를 판단해 유효함을 표시
    비밀번호 확인은 비밀번호와 같을 때 유효함을 표시
    이메일은 정규표현식에 일치할 때 인증번호를 보내는 버튼 활성화

    인증번호 전송을 누르면 인증번호를 보내는 중임을 알리는 팝업이 뜨고 아래 인증번호를 입력하는 칸 활성화
    javamail과 AWS SES를 이용해 메일 전송
    (Gmail SMTP 서버를 사용하려 했지만 EC2 환경에서는 SES SMTP 사용이 권장되어서 변경)

    전송한 인증번호가 입력된 경우 유효함이 표시되고
    폼에 변경이 있을 때마다 검사해 모든 입력칸이 유효할 때 계정 생성 버튼 활성화
    계정이 생성되면 로그인 페이지로 이동
  • 전체 게시글, 댓글수, 조회수 표시
    • 게시글 정보와 함께 댓글수, 조회수 표시
    넘겨받은 게시글 목록을 jstl의 반복문으로 나타냄
  • 검색

    • Mybatis의 if 동적 쿼리로 조건에 맞는 게시글을 가져옴
    조건(condition)과 검색단어(keyword)의 초기값은 title, 빈 문자열이며 전체 게시글을 가져옴
    Criteria 객체는 게시글 목록을 가져오는 getBoardList 함수의 매개변수로
    클릭한 페이지, 검색 조건이 담겨있음
  • 페이징
    • 한 페이지 당 10개씩의 게시글을 표시
    컨트롤러에서 조건이 담긴 Criteria를 가지고 페이징에 필요한 변수들을 가진 Paging 객체를 만듦
    MySQL의 LIMIT를 사용해 startPage부터 10개씩 가져옴
    startPage를 포함한 필요한 변수들은 Paging 객체에서 전체 게시글 수인 totalCount 를 바탕으로 계산
    이전, 다음 버튼의 활성화 여부를 위한 prev, next 변수도 계산
    조건에 따라 페이징 버튼 표시
  • 스프링 시큐리티의 요청 인터셉트
    • 로그인 여부에 따라 페이지 접근을 제어하기 위해 사용
    insert, delete, update 로 시작하는 요청을 제외한 나머지는 모두 비인증 상태에서 접근 가능

    커스텀 로그인페이지 경로 설정(login-page), 로그인 후 게시글 화면으로 이동(default-target-url)
    로그인에 실패하면 다시 로그인 페이지를 보이게 함(authentication-failure-url)

    로그아웃 후 게시글 화면으로 이동(logout-success-url)
    또한 허가되지 않은 접근인 경우 에러 페이지로 이동

    데이터베이스에서 유저 정보를 가져오기 위해
    스프링 시큐리티의 UserDetailsService를 구현한 CustomUserDetailsService를 만들고 등록

    CustomUserDetailsService에서는 주어진 username으로 유저 정보를 가져오고
    userDetails 객체에 담아 리턴
  • 게시글 CRUD
    • 글쓰기 폼에는 ckeditor를 적용


    writer가 Authentication에 저장된 username과 같으면 글쓰기를 진행
    수정, 삭제 요청도 같은 과정을 거침

    Authentication에 저장된 유저 정보의 username을 가져와서
    현재 게시글 페이지의 writer와 같으면 수정, 삭제 버튼을 표시
  • 댓글 CRUD
    • 게시글을 가져오면서 게시글 번호에 해당하는 댓글 목록을 가져옴
    대댓글은 부모댓글의 아래에 순서대로 표시되고,
    일반 댓글에만 대댓글을 달 수 있게 버튼을 표시

    게시글과 마찬가지로 writer가 Authentication에 저장된 username과 같은 경우에만
    수정, 삭제를 진행 수정, 대댓글을 클릭하면 바로 아래에 textarea와 버튼을 나타내서 Form 전송이 가능해 짐
  • 클라우드에 프로젝트 업로드
    • 서버가 계속 동작하도록 AWS EC2에 프로젝트를 업로드함 자바, 톰캣 설치 후 WinSCP 프로그램으로 프로젝트 war 파일을 EC2의 톰캣 폴더에 옮김
